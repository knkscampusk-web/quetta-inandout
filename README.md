# QUETTA 일일 등록 보고 도구

경영지원 자료(일일등록명단 + 마스터)를 넣으면 등록파일 `2026raw정규/반수` 시트에
붙여넣을 행을 산출하는 팀 공유 도구입니다. **GitHub Pages + Firebase Firestore.**

## 들어가는 자료 (매일)
1. **일일등록명단** (경영지원) — 그날 등록·환불·신규·장학
2. **마스터(원생마스터 시트 포함)** — 수험번호 → 성별·반 매핑 + 재원 모수
3. **현황표 숫자** (선택) — 총인원·미납 검증용

## 나오는 결과
- 화면 대시보드 (전형/반별 등록·남여·신규·환불·장학, 전일 대비 증감, 미등록)
- `2026raw정규/반수` 붙여넣기 행 → **복사** 또는 **엑셀 다운로드**
- raw만 붙여넣으면 일자별확인 · 2026정규 · 일일보고가 자동 계산됨

## 개강 전/후
- 설정의 **개강일** 기준 자동 분기
- 개강 전: 전형·계열 단위 / 개강 후: 학생 **반(마스터 매핑) 단위**

---

## 배포 (한 번만)

### 1) Firebase
1. console.firebase.google.com → 프로젝트 생성
2. **Authentication → 로그인 방법 → 익명(Anonymous)** 사용 설정
3. **Firestore Database** 생성
4. **규칙** 탭에 `firestore.rules` 내용 붙여넣고 게시
5. 프로젝트 설정 → 웹 앱 추가 → `firebaseConfig` 객체 복사
6. ⚠ **API 키 → HTTP 리퍼러 제한**: GitHub Pages 도메인만 허용

### 2) GitHub Pages
1. 저장소에 `index.html` 업로드
2. Settings → Pages → 브랜치 지정 → 게시
3. (Netlify 안 씀)

### 3) 첫 접속 설정
1. 게시된 주소 접속 → 게이트에서 **초기 설정** 클릭
2. **Firebase 연결**: `firebaseConfig` JSON 붙여넣기 → 저장 & 재접속
3. **팀 비밀번호** 설정 (해시로만 저장)
4. **개강일** 입력 (정규/반수)
5. 팀원에게 주소 + 비번 공유

> 배포본에 config를 고정하려면 `index.html` 상단 `BUILTIN_CONFIG`에 직접 넣어도 됩니다.

---

## 보안 메모
- 규칙은 `request.auth != null` (익명 로그인 세션) 기준. **`allow read, write: if true` 안 씀.**
- 게이트 비번은 앱단 1차 잠금(해시 저장). 더 강한 통제가 필요하면 Firebase Auth 이메일 허용목록으로 승격 권장.
- 학생 개인정보가 들어가므로 API 키 도메인 제한은 필수.

## 알려진 한계 (v1)
- **반수 시즌**: 정규 기준으로 만들었고 구조는 동일. 반수 일일파일/현황표 샘플 주시면 같은 틀로 검증.
- **단과**: 다음 단계. 동일 구조로 확장 예정.
- **환불자 성별**: 환불 학생이 정규 마스터에 없으면(반수/편입) 성별 미상으로 경고 표시 → 해당 시즌 마스터를 함께 넣으면 해소.
- **접수/합격/편성인원**: 등록 자료엔 없는 값이라 raw에서 비움(접수 측 보고는 별도).
